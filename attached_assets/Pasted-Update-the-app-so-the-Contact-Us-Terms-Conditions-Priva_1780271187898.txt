Update the app so the **Contact Us, Terms & Conditions, Privacy Policy, and About Us** pages use the same endpoint/data behavior as the legacy iOS app.

## Legacy behavior to preserve

### Contact Us

This does **not** come from an API endpoint.

The old app uses a hardcoded support email address:

* `udecidemobileapplication@gmail.com`

Implement Contact Us so it:

* shows the support email address
* opens the user’s mail client with a `mailto:` link
* does **not** call the backend for this page

### Terms & Conditions, Privacy Policy, About Us

These pages come from the legacy backend endpoint:

* `POST /static_pages`

The request must send a field named:

* `page_code`

Use these exact legacy values:

* `aboutus`
* `privacypolicy`
* `termsconditions`

Map them like this:

* About Us → `page_code=aboutus`
* Privacy Policy → `page_code=privacypolicy`
* Terms & Conditions → `page_code=termsconditions`

The backend response should be treated as returning content like:

* `pageTitle`
* `pageContent`

Render those values in the app page view.

## What to implement

1. Centralize the endpoint definitions for static/legal pages.
2. Add a reusable API method for:

   * `POST /static_pages`
3. Add a helper function like:

   * `getStaticPage(pageCode)`
4. Add page-specific loaders:

   * `getAboutUs()`
   * `getPrivacyPolicy()`
   * `getTermsAndConditions()`
5. Make sure the UI routes the correct page to the correct `page_code`.
6. Do not hardcode the HTML/body for Terms, Privacy, or About in the frontend.
7. Render the returned content from the backend response.
8. Support HTML or rich text content in `pageContent` if the backend returns formatted content.
9. Add comments explaining that these are legacy backend-managed static pages.
10. Keep Contact Us separate from `/static_pages`, because it is email-based, not API-based.
