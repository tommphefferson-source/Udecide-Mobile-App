## Functional requirements

### 1) Elections

The old iOS app directly called Google Civic:

* `https://www.googleapis.com/civicinfo/v2/elections`
* `https://www.googleapis.com/civicinfo/v2/voterinfo`

Implement:

* `GET /api/elections`
* `GET /api/elections/voter-info`

Behavior:

* call Google Civic API
* use environment variable `GOOGLE_CIVIC_API_KEY`
* accept address as a query param
* for voter info also accept `election_id`
* normalize response into clean app-facing JSON
* include election list, contests, state election administration info, and relevant URLs when available

### 2) Polls

Old endpoints:

* `/poll_listing`
* `/poll_results`
* `/polling`

Implement:

* `GET /api/polls`
* `POST /api/polls/{poll_id}/vote`
* `GET /api/polls/{poll_id}/results`

Requirements:

* back these with local mock data initially
* support listing polls
* support submitting a vote
* support returning poll results
* store vote submissions in-memory for now unless the chosen stack has a lightweight better option
* return realistic JSON

### 3) Home data

Old endpoint:

* `/home_data`

Implement:

* `GET /api/home`

Return:

* app home metadata
* `news_url`
* any other useful homepage metadata from a local data file or equivalent

### 4) News

The old app mainly used a `news_url` returned by home data.

Implement:

* `GET /api/news`

Behavior:

* return a structured response that includes:

  * `news_url`
  * `title`
  * `description`
  * `source`
* back this from local data for now
* keep the response expandable for future live article integrations

### 5) Civics / quizzes / questionnaires

Old endpoints:

* `/quiz_menu_list`
* `questionaires_listing`
* `update_quiz_log`
* `/political_terminology_list`
* `/get_parties`

Implement:

* `GET /api/quizzes`
* `GET /api/questionnaires`
* `POST /api/quizzes/progress`
* `GET /api/terminology`
* `GET /api/parties`

Requirements:

* use local data files or equivalent mock-backed storage
* include quiz categories/menu
* include questionnaire questions
* include terminology items
* include party reference data
* allow quiz progress submission and return a success response
